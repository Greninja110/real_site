A Practical Guide to Parameter Tuning for KNN Using the Iris Dataset

Neural networks are at the heart of many breakthrough technologies today, from image recognition to language translation. But what exactly are they, and how do they work? In this beginner-friendly guide, I'll break down the core concepts of neural networks in simple terms.

What is Parameter Tuning?

Parameter tuning (also known as hyperparameter tuning) is the process of systematically adjusting the settings (parameters) of machine learning algorithms to optimize their performance. These parameters are not learned directly from the data; rather, they are set manually or through automated methods before training the model.

Think of parameters as "knobs" or "settings" you can adjust to make the algorithm perform better. Different combinations can significantly impact accuracy, efficiency, and the generalization of your model to new, unseen data.

Why is Parameter Tuning Important?

  1. Performance Improvement: Proper tuning often results in higher accuracy and better predictive performance.
  2. Avoiding Overfitting: Helps models generalize better to unseen data by controlling complexity.
  3. Efficiency: Finds the balance between accuracy and computational cost, making the model practical for real-world use.

Common Methods of Parameter Tuning:

  1. Grid Search: Systematically explores a defined set of parameters.
  2. Randomized Search: Randomly samples parameter combinations within a specified range.
  3. Bayesian Optimization: Iteratively refines parameter choices based on past evaluations.

What is the Iris Dataset?

The Iris dataset is one of the most popular and widely-used datasets in machine learning, originally introduced by Ronald A. Fisher in 1936. It contains measurements of 150 iris flowers from three different species:

  1. Iris Setosa
  2. Iris Versicolor
  3. Iris Virginica

Dataset Features:

The dataset includes four numerical features:

  • Sepal Length (in centimeters).
  • Sepal Width (in centimeters).
  • Petal Length (in centimeters).
  • Petal Width (in centimeters)..

Why is it Popular?

The Iris dataset is commonly used for machine learning tutorials and benchmarking because:

  • It's small and manageable, ideal for beginners.
  • Clearly demonstrates concepts of classification and parameter tuning.
  • Helps evaluate the performance of different algorithms.
  • Widely used for demonstrating machine learning algorithms.

What is the K-Nearest Neighbors (KNN) Algorithm?

K-Nearest Neighbors (KNN) is a simple yet effective supervised machine learning algorithm used for both classification and regression tasks. It classifies new data points based on the "nearest" existing data points in the feature space.

How Does KNN Work?

  • Step 1: Select the number of neighbors (k).
  • Step 2: Calculate the distance (e.g., Euclidean, Manhattan) from the new data point to all other points in the dataset.
  • Step 3: Find the k nearest points (neighbors).
  • Step 4: Assign the new data point the class most common among these neighbors (for classification tasks) or average their values for regression.

Key Points:

  • Simple Easy to understand and implement.
  • Lazy learner: Does not explicitly learn a model; it stores training data and makes decisions based directly on it.
  • Distance-Based: Sensitive to the scale of data (often requires feature scaling).
  • Parameter Tuning: Performance greatly depends on choosing an optimal value of k and appropriate distance metrics.

Parameter Tuning in KNN

When tuning a k-Nearest Neighbors (kNN) algorithm, you’re essentially trying to find the best combination of hyperparameters to maximize your model’s performance. Below is a detailed explanation of the key parameters you can tune, along with typical value ranges and considerations:

1. Number of Neighbors (k)

  • Definition: The parameter k specifies how many neighbors (closest points) to look at when making a prediction.
  • Typical Values: Common choices range from 1 to 20, but you might consider a broader range depending on the dataset. A good rule of thumb is to start with an odd number (especially for binary classification) to avoid ties.
  • Considerations:
    1. Small k: Leads to models that capture noise (high variance) and may overfit.
    2. Large k: May overly smooth the decision boundary, increasing bias.
    3. Tuning Strategy: Use cross-validation to select the optimal value. 

                                                            import matplotlib.pyplot as plt
                                                            import numpy as np
                                                            import pandas as pd
                                                            from sklearn.datasets import load_iris
                                                            from sklearn.model_selection import train_test_split
                                                            from sklearn.preprocessing import StandardScaler
                                                            from sklearn.neighbors import KNeighborsClassifier
                                                            from sklearn.metrics import accuracy_score
                                                            
                                                            # Load the Iris dataset
                                                            iris = load_iris()
                                                            X = iris.data
                                                            y = iris.target
                                                            
                                                            # Define the test sizes and the list of k values to evaluate
                                                            test_sizes = np.arange(0.01, 1.0, 0.01)
                                                            k_values = [1, 2, 3, 4, 5]
                                                            
                                                            # Dictionaries to store results:
                                                            # For plotting: results[k] = (list of test sizes, list of accuracies)
                                                            results = {k: ([], []) for k in k_values}
                                                            
                                                            # Also, store all accuracies keyed by test size for CSV output
                                                            all_results = {}
                                                            
                                                            # Loop over each test size (outer loop)
                                                            for test_size in test_sizes:
                                                            # Split the dataset using the current test size (same split for all k)
                                                            X_train, X_test, y_train, y_test = train_test_split(
                                                                X, y, test_size=test_size, random_state=42
                                                                )
                                                                
                                                                # Dictionary to store the accuracy for each k at this test size
                                                                row_results = {}
                                                                
                                                                # For each k value, compute accuracy if enough training samples exist
                                                                for k in k_values:
                                                                if len(y_train) < k:
                                                                # Not enough samples to train with this k value
                                                                row_results[k] = None
                                                                else:
                                                                # Standardize the features
                                                                scaler = StandardScaler()
                                                                X_train_scaled = scaler.fit_transform(X_train)
                                                                X_test_scaled = scaler.transform(X_test)
                                                                
                                                                # Create and train the k-NN classifier with current k value
                                                                knn = KNeighborsClassifier(n_neighbors=k)
                                                                knn.fit(X_train_scaled, y_train)
                                                                
                                                                # Make predictions and calculate accuracy
                                                                y_pred = knn.predict(X_test_scaled)
                                                                accuracy = accuracy_score(y_test, y_pred) * 100  # in percentage
                                                                row_results[k] = accuracy
                                                                
                                                                # Save results for plotting
                                                                results[k][0].append(test_size)
                                                                results[k][1].append(accuracy)
                                                                
                                                                all_results[test_size] = row_results
                                                                
                                                                # Print the accuracies for the current test size
                                                                accuracy_str = ", ".join(
                                                                    [f"k={k}: {row_results[k]:.2f}%" if row_results[k] is not None else f"k={k}: N/A"
                                                                    for k in k_values]
                                                                    )
                                                                    print(f"Test size: {test_size:.2f} -> {accuracy_str}")
                                                                    
                                                                    # Create a DataFrame for CSV output
                                                                    # Each row: test_size, accuracy for k=1, k=2, ..., k=5
                                                                    data_rows = []
                                                                    # Sort test_sizes to have a consistent order in the CSV
                                                                    for test_size in sorted(all_results.keys()):
                                                                    row = {'test_size': test_size}
                                                                    for k in k_values:
                                                                    row[f'accuracy_k{k}'] = all_results[test_size][k]
                                                                    data_rows.append(row)
                                                                    
                                                                    df = pd.DataFrame(data_rows)
                                                                    df.to_csv('p6(adv).csv', index=False, float_format='%.3f')  # Rounds floats to 3 decimal places
                                                                    
                                                                    # Plot all curves on a single graph with different colors for each k value
                                                                    plt.figure(figsize=(10, 6))
                                                                    colors = ['b', 'g', 'r', 'c', 'm']  # Colors for different k values
                                                                    
                                                                    for idx, k in enumerate(k_values):
                                                                    ts_list, acc_list = results[k]
                                                                    plt.plot(ts_list, acc_list, marker='o', linestyle='-', color=colors[idx],
                                                                    label=f'k = {k}')
                                                                    
                                                                    plt.xlabel('Test Size (Fraction of Dataset)')
                                                                    plt.ylabel('Accuracy (%)')
                                                                    plt.title('Test Size vs. Accuracy for k-NN on Iris Dataset for Different k Values')
                                                                    plt.legend()
                                                                    plt.grid(True)
                                                                    plt.show()

2. Distance Metrics

  • Definition: The distance metric defines how “close” or “similar” data points are.
  • Common Choices:
    1. Euclidean Distance (L2 norm): Standard metric for continuous variables.
    2. Manhattan Distance (L1 norm): Sometimes works better when features are not on the same scale or when outliers are present.
    3. Minkowski Distance: A generalization that includes both Euclidean (when p = 2) and Manhattan (when p = 1).
    4. Other Metrics: In some cases, you might experiment with Chebyshev distance or even custom distance functions depending on the domain.
  • Considerations:
    1. The choice of metric can greatly affect model performance.
    2. Scale your features appropriately (e.g., using standardization) so that one feature does not dominate the distance calculation.

3. Weight Function

  • Definition: Determines how much influence each neighbor has on the final prediction.
  • Options:
    1. Uniform: All neighbors contribute equally to the prediction.
    2. Distance: Closer neighbors have a higher weight, which can be useful when points further away are less reliable.
  • Considerations:
    1. In classification, distance weighting can help smooth out predictions in regions where the density of points varies.
    2. You can often test both options to see which performs better with cross-validation.

4. Algorithm Used for Computing Neighbors

  • Definition: Specifies the underlying algorithm to compute nearest neighbors efficiently.
  • Options:
    1. Brute Force: Computes distances from every training point to the query point; can be computationally expensive on large datasets.
    2. KD Tree: More efficient for low-dimensional data but less effective in high dimensions.
    3. Ball Tree: Can be more efficient than KD Trees in certain conditions, particularly with higher dimensions.
  • Considerations:
    1. The choice of algorithm may depend on dataset size and dimensionality.
    2. Sometimes, performance differences are minor, but memory and speed considerations can be critical.

5. Leaf Size (for tree-based methods)

  • Definition: Affects the speed of the tree-based algorithms (KD Tree or Ball Tree) and memory consumption.
  • Typical Values:
    1. A common range is between 20 and 50. However, this might be adjusted based on your specific dataset and hardware constraints.
  • Considerations:
    1. Smaller leaf sizes may result in a more detailed tree but slower query times.
    2. Larger leaf sizes may speed up the queries but can lose some fine-grained distinctions in the data.

6. Parameter p in Minkowski Distance

  • Definition: The parameter p in the Minkowski distance metric allows you to adjust the “power” of the norm.
  • Typical Values:
    1. p = 1 corresponds to Manhattan distance.
    2. p = 2 corresponds to Euclidean distance.
  • Considerations:
    1. Experimenting with different values of p can allow you to find a distance measure that best captures the structure of your data.

Tuning Strategy

  • Cross-Validation:Use grid search or randomized search with cross-validation to explore the combination of these parameters. This helps in understanding the trade-offs between bias and variance.
  • Scaling:Remember to standardize or normalize your features before applying kNN, as distance metrics are sensitive to the scale of the data.
  • Performance Metrics:Choose appropriate performance metrics (accuracy, F1-score, RMSE, etc.) based on whether your problem is classification or regression.

Conclusion

By systematically exploring different combinations of these parameters, you can significantly enhance the performance of your kNN model. Often, tuning these parameters in combination rather than in isolation is key to achieving the best results for your specific application.
